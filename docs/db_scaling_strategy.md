Физическую репликацию следует использовать для осуществления отказоустойчивости БД и при большом количестве SELECT запросов.

Преимуществами являются простота настройки, обеспечение отказоустойчивости, снижение нагрузки на основную БД при SELECT запросах

Недостатками является большое количество памяти для копирования полной БД, отсутствие изменение нагрузки для INSERT и UPDATE запросов

Можно применить ко всей БД для отказоустойчивости. SELECT запросы происходят с реплики, снижая нагрузку на основную БД. Eventual consistency будет достаточно для нашего проекта

Логическую репликацию следует использовать для осуществления отказоустойчивости БД. В отличие от физической логическая дает больше свободы для распределения данных.

Преимущества: гибкость настройки, обеспечение отказоустойчивости,

Недостатки: сложность настройки

Можно выборочно сделать репликацию ключевых аспектов БД для сохранения памяти или упрощения анализа данных. Сложно реализовать, требует четкого понимания, что нужно для анализа данных

Шардирование позволит распределить нагрузку на БД полностью

Преимущества: возможность распределение всех запросов на разные сервера БД

Недостатки: сложность реализации, усложнение логики

Можно распределить users, products, reviews и orders, order_items на два разных сервера.
Это немного улучшит производительность запросов с небольшим усложнением в логике работы.
